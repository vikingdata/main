MySQL: Stored Procedures INOUT

MySQL stored procedures are easy to understand once you follow the simple rules for IN and OUT.
  1. IN rquires there to be an input. If you define a default, you don't have to submit inout.
  2. OUT requires you to have output in the charater type defined. HINT: you should force a conversation or possibly error out.
  3. INOUT requires and input and it prints an output.
For more information read:
  1. https://dev.mysql.com/doc/refman/8.0/en/create-procedure.html

MySQL Stored Procedures

Create these stored procedure in mysql.

Simple function. 

DELIMITER //
CREATE PROCEDURE now()
    BEGIN
    SELECT *  FROM products;
    END //

DELIMITER ;

-- Simple procedure. It takes a number and adds 10. 

DELIMITER //
CREATE PROCEDURE sp_i(IN no int4)
  BEGIN
  select no + 10;
  END //
DELIMITER ;


--  Simple procedure, adds one day to time.
DELIMITER //
  CREATE PROCEDURE sp_day(out date1 datetime)
    BEGIN
    select now + 1 days ;
    END //
DELIMITER ;

--  Simple procdure, adds two numbers together.

DELIMITER //
  CREATE PROCEDURE sp_iio(in no1 int, In no2 int, OUT sout int)
    BEGIN
    DECLARE s int DEFAULT 0;
    select s;
    END //
DELIMITER ;

-- another way
DELIMITER //
CREATE PROCEDURE sp_ioi(inout inital int, In no2 int)
  BEGIN
  let initial = intitial + no2;
  select initial;
  END //
DELIMITER ;

-- simple one inout variables

DELIMITER //
  CREATE PROCEDURE sp_io(inout no int)
    BEGIN
    select no + 10;
    END //
DELIMITER ;


-- simple one inout variables, with error

DELIMITER //
  CREATE PROCEDURE sp_errorcheck(inout no int)
BEGIN
    IF no is NULL then set no = -1; END IF; 
    select no;
    END //
DELIMITER ;
