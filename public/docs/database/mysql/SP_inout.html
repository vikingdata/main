MySQL: Stored Procedures INOUT

MySQL stored procedures are easy to understand once you follow the simple rules for IN and OUT.
  1. IN requires there to be an input.
  2. OUT requires you to have output in the charater type defined. HINT: you should force a conversation or possibly error out.
  3. INOUT requires and input and it prints an output.

Remember these important tips. Lets take the variable VAR1. It can be confusing to remmeber when to use "@VAR1" or "VAR1" at times.

  1. Whenever you "read" or "select" the variable VAR1 you must use "@VAR1"". This applies to the mysql shell or in stored procedures.
  2. Whenever you set VAR1 in the mysql shells, you must use "@VAR1".
  3. Whenevr you set, or declare, or use in proc_parameters (IN, OUT, INOUT) VAR1 in a stored procedure, you must use "VAR1".

IN example

IN variables can be static or environment variables. They are only used in the stored procedure. If any evironment variable is used, it is not changed when the stored procedure ends.

OUT and INOUT example

The important thing to remember about OUT defined variables are:
  1. If the variable in the mysql environment exists, it is set to NULL.
  2. A variable must be submitted to an OUT defined input.
  3. A variable submitted to an OUT input variable will be references and NOT copied. This means any changes to the variable will appear in the envionment submitted to the procedures.

For OUT, We see the @i1 variable is submitted as "1" to the stored procedure. We see the sp immediately makes it NULL. At the end it changes it to "3" and the environment variables @i1 becomes "3".

For INOUT, @i2 is submitted as "2". It does not change upon entering. It s changed to "4" and then the environment variable is changed.
